# 应用上传工具使用指南

## 🚀 功能说明
这个工具可以让你直接在网页上上传 HTML 应用，自动部署到站点上，不需要手动写代码和提交。

## 📋 使用步骤

### 1. 创建 GitHub Personal Access Token
首先需要创建一个有 repo 权限的 GitHub Token：
1. 登录 GitHub → 点击头像 → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. 点击 "Generate new token" → 选择 "Generate new token (classic)"
3. 填写 Note，比如 "WebSnack Uploader"
4. 勾选 `repo` 权限（完整的仓库读写权限）
5. 点击 "Generate token"
6. 复制生成的 Token（只显示一次，保存好）

### 2. 填写应用信息
- **应用分类**：选择游戏/工具/应用
- **应用名称**：你的应用名称，比如 "贪吃蛇游戏"
- **应用描述**：简单描述应用功能
- **应用图标**：选择一个合适的 emoji 图标

### 3. 上传文件
- **index.html**（必需）：应用的主 HTML 文件
- **style.css**（可选）：样式文件，会自动注入到 HTML 中
- **main.js**（可选）：脚本文件，会自动注入到 HTML 中
- **其他资源**（可选）：图片、字体等资源文件，会保存到应用的 assets 目录下

### 4. 预览（可选）
点击 "预览效果" 按钮，可以在页面右侧预览应用的实际效果，确认无误后再提交。

### 5. 提交部署
1. 输入你创建的 GitHub Token
2. 点击 "上传并部署" 按钮
3. 等待部署完成（大约 30 秒）
4. 部署成功后会显示应用访问链接

### 6. 等待自动部署
提交后 GitHub Actions 会自动构建和部署，大约 1-2 分钟后就可以在线访问你的应用了！

## ✨ 特性
- ✅ 自动生成规范的应用目录结构
- ✅ 自动注入返回首页链接和公共样式
- ✅ 自动添加应用到首页导航
- ✅ 支持预览效果，确认后再部署
- ✅ 自动提交到 GitHub 仓库
- ✅ 触发自动部署流程

## 🔒 安全说明
- GitHub Token 仅在浏览器中使用，不会上传到任何服务器
- Token 只用于提交文件到你的仓库，不会被存储
- 建议使用后撤销 Token，确保账户安全

## 📝 注意事项
1. HTML 文件中不要使用绝对路径引用资源，所有资源都要相对路径
2. 如果应用需要引用外部 CDN 资源，请确保使用 HTTPS 链接
3. 单文件 HTML 应用可以只上传 index.html
4. 资源文件大小建议不要超过 5MB，太多大文件会影响加载速度
5. 如果应用比较复杂，建议还是手动创建目录进行开发

## ❓ 常见问题

### Q: 部署后应用访问 404？
A: 等待 1-2 分钟让 GitHub Pages 部署完成，然后刷新页面。

### Q: 应用样式显示不正常？
A: 检查 CSS 文件是否正确上传，路径引用是否正确。预览时如果正常，部署后也应该正常。

### Q: Token 权限错误？
A: 确认 Token 已经勾选了 `repo` 权限，并且没有过期。

### Q: 提交失败提示权限不足？
A: 确认你对仓库有写入权限，Token 正确有效。
