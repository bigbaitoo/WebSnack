# 应用上传工具使用指南

## 🚀 功能说明
这个工具可以让你直接在网页上上传 HTML 应用，保存到本地浏览器中立即使用，不需要 GitHub Token，也不需要部署。

## 📋 使用步骤

### 1. 填写应用信息
- **应用分类**：选择游戏/工具/应用/俄语专区
- **应用名称**：你的应用名称，比如 "贪吃蛇游戏"
- **应用描述**：简单描述应用功能
- **应用图标**：选择一个合适的 emoji 图标

### 2. 上传文件
- **index.html**（必需）：应用的主 HTML 文件
- **style.css**（可选）：样式文件，会自动注入到 HTML 中
- **main.js**（可选）：脚本文件，会自动注入到 HTML 中
- **其他资源**（可选）：图片、字体等资源文件，会保存到应用的 assets 目录下

### 3. 预览（可选）
点击 "预览效果" 按钮，可以在页面右侧预览应用的实际效果，确认无误后再保存。

### 4. 保存到本地
点击 "保存到本地" 按钮，应用会保存在浏览器的本地存储中，你可以立即打开使用。

### 5. 导出应用（可选）
点击 "导出应用包" 按钮，可以将应用导出为单个 HTML 文件，分享给其他人或者保存到本地。

## ✨ 特性
- ✅ 不需要 GitHub Token，不需要任何权限
- ✅ 自动生成规范的应用结构
- ✅ 自动注入返回导航
- ✅ 支持预览效果，确认后再保存
- ✅ 应用保存在浏览器本地，随时可以访问
- ✅ 支持导出为单个 HTML 文件，方便分享

## 🔒 数据存储说明
- 所有上传的应用都保存在浏览器的本地存储（localStorage）中
- 不会上传到任何服务器，完全本地运行
- 清除浏览器数据会导致应用丢失，重要应用建议导出备份
- 应用只在当前浏览器中可用，其他设备需要重新上传

## 📝 注意事项
1. HTML 文件中不要使用绝对路径引用资源，所有资源都要相对路径
2. 如果应用需要引用外部 CDN 资源，请确保使用 HTTPS 链接
3. 单文件 HTML 应用可以只上传 index.html
4. 资源文件大小建议不要超过 5MB，太多大文件会影响加载速度
5. 建议定期导出重要的应用进行备份

## ❓ 常见问题

### Q: 应用会丢失吗？
A: 只要不清除浏览器数据，应用会一直保存在本地。建议定期导出备份。

### Q: 可以在其他设备上访问吗？
A: 不可以，应用保存在当前浏览器中，其他设备需要重新上传。

### Q: 可以分享给其他人吗？
A: 可以，使用导出功能导出为单个 HTML 文件，发给其他人直接打开就可以用。

### Q: 应用存储上限是多少？
A: 浏览器 localStorage 一般有 5-10MB 的存储空间，建议不要存储太多大体积应用。

### Q: 如何永久部署到网站上？
A: 如果需要永久部署到站点上，可以将导出的 HTML 文件发给管理员，或者使用 Git 提交到仓库。
